Procedure for registering a project with USAP-DC

For frequently asked questions see also FAQ page

Overview

Registering your project with the USAP-DC Project Catalog is a very straightforward process. You (the submitter) will fill out the Project Registration Form with the required information. Throughout the process you may save your progress and return to it at any point. You may even make edits to the page after it has been processed. Once submitted you will receive an email notification of a completed form submission. This will then be reviewed by USAP-DC Curators, who will check over the Award Numbers, listed Investigators and Collaborators, and more. This review process usually takes 1-2 business days. Curators may contact you directly with questions regarding your submission. Once the process is finished, you will be notified by the USAP-DC Curators via email.

The USAP-DC Project Landing Page will host all project related information, citations, datasets, metadata, and more. PIs and Collaborators will have the option to Edit the page throughout its lifetime with updates and modifications. The USAP-DC Curators will process all of those changes as stewards of the archive. The goal is long-term project and research data preservation.

Why register your project?

NSF guidelines require that PIs who receive an award from the NSF U.S. Antarctic Program register their project with the USAP-DC Project Catalog using their award number. Related award numbers for collaborative research projects are also required. Projects awarded during 2020 and later are required to include USAP-DC Project Catalog links in their Year-One Annual Reports. Full datasets, metadata files, and derived data products must be deposited in a long-term and publicly accessible archive.

Registering Your Project

USAP-DC provides a simple web form which asks you to fill out basic project information including co-Investigators, Data Managers, Project Summary, field location, external datasets, publications, and more.

Once you’ve logged into www.usap-dc.org/submit/project, you may select the associated NSF Award from a dropdown menu at the top of the form. Once selected, the Award will pre-populate the submission form with the information available from the NSF Fastlane Award Database.

For Collaborative Research Projects, use the “Add additional award linked to project” feature to add the Award Numbers for the different institutions. This feature is located beneath the dropdown menu and does not have a limit to how many award numbers that can be included. We strongly recommend the primary institution fill out the Project Registration form. Before moving on, make sure that all individuals associated with the award are included in the registration form. To include anyone not automatically added via the Award Number auto-population, use the “Add another person” option beneath the “Additional Personnel” field. Type the person’s first and last name into the field. This input also functions as a search to see if the individual is already registered within the USAP-DC system. If the individual’s name pops up, click on the name and it will auto-populate the rest of the name field with what is currently on file.

Large Programs

If your project/award is part of a larger, overarching science program (e.g. ANDRILL, COLDEX, Thwaites-ITGC, etc.), please select the program from the Special Science Program dropdown menu. If it is not listed, please make a note of this for the USAP-DC Staff in the comment box at the end of the webform. This label will allow users to search and filter for projects that are flagged as part of these larger initiatives.

Keywords

You can select Location and GCMD keywords from pre-populated lists. These keywords are based on the GCMD keyword vocabulary and will be used to enter your project in the Antarctic Metadata Directory.

Linking Datasets

You can include datasets archived in external repositories, or within USAP-DC. If datasets are not yet available, they can be added at a later time using the “Edit” button on your Project Landing Page.

Datasets submitted to USAP-DC will automatically be linked to your project page via the NSF Award Number. Externally archived datasets require a link to where the dataset may be accessed.

Publications & References

Linking publications and references is easy, and can be done at any time. Adding a DOI and/or URL (when available) is simple and easy. You may also click the “Search Crossref for publications associated with linked awards” button.

After Submission

Once you submit your project registration form, you will receive a confirmation email. Your project page will not automatically be available on the website. USAP-DC Staff will check over your submission before your project page is published. This usually takes 1-2 business days. During this review, Staff may contact you if we have any questions regarding your submission.

AMD Registration

Using the project information provided by you in the USAP-DC registration forms, we will create a metadata record in the required format (DIF) and submit it to the Antarctic Metadata Directory (AMD) on your behalf. This AMD registration is required for the final project report to NSF, and will usually be done after the initial datasets have been linked to the project page. These submissions can be updated.

Revisions & Updates

You can update the project landing page at any time using the “Edit” button, located in the top-right corner of the page, or clicking the “Edit” button next to the project listed in your User Dashboard.

The Edit function takes you to the same pre-populated form with all the information currently available for the project. Changes can be made to any of these fields at any time, saved, and submitted just as you did for the initial submission. Once these updates are submitted, you will receive a confirmation email notification regarding these edits. However, these changes will not automatically be visible on the website. Edits will be reviewed by a USAP-DC Curator before they are published on the project page.

Please note, only users linked to the project, via ORCID or email address, can access the Edit forms. If an un-linked user (or for whom the authentication on file is insufficient) tries to access this feature, the user will be presented with an option to request access.
